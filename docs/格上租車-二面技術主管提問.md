## A. 技術架構與 Google 整合深度

### A1. Google 生態整合
> 職缺特別提到 Google ADK / Vertex AI，這在台灣企業中並不常見

- 目前團隊與 Google Cloud 的合作程度到哪裡了？是已經有 Vertex AI 的 production workload 在跑，還是正在從其他雲（如 AWS/Azure）遷移過來？
- 選擇 Google ADK 作為 Agent 框架的主要考量是什麼？有評估過 LangGraph 或其他開源方案嗎？還是希望進來的人一起做技術選型？
- LLM 的選用上，是以 Gemini 系列為主，還是也會用到 OpenAI / Anthropic / 開源模型？有沒有 multi-model 的策略？

### A2. AI Agent 架構現況
> 對應職缺描述：任務規劃、記憶管理、工具調用與流程執行

- 「智能移動 AI 代理人」目前的開發階段大概在哪裡？是已經有 MVP 在內部測試，還是處於架構設計階段？
- Agent 的記憶管理目前是怎麼做的？有用到 Vector DB 嗎？如果有，是 Vertex AI Vector Search 還是其他方案（Pinecone / Weaviate）？
- Function Calling / Tool Use 目前串接了哪些外部系統？（租車系統、LINE GO、交通 API、餐飲訂位等）

### A3. RAG 架構
> 職缺提到「深度優化 RAG 架構，解決模型幻覺問題」

- 目前 RAG 面臨的最大挑戰是什麼？是檢索品質（recall/precision）、還是生成端的幻覺、還是知識庫的更新頻率？
- 知識庫的範圍大概涵蓋哪些領域？（租車條款、車輛規格、交通資訊、長照服務...）
- 有沒有考慮或已經在做 Evaluation 的自動化？像是 RAGAS / DeepEval 那類的框架？

## 提問策略建議

1. **開場先從 A2（Agent 現況）切入** — 這是你最強的專業領域，也最能引發深度技術討論
2. **C1（3~6 個月 roadmap）必問** — 直接影響你對 offer 的判斷
3. **B1（AI coding 工具）適合穿插聊** — 體現你是高效工程師，也能觀察團隊的工程文化成熟度
4. **D2（主管期待）放最後問** — 留下一個好印象，表現你在乎的是做好事情而不只是技術
5. **不要全問** — 挑 5~7 題，根據現場氛圍和前面討論的脈絡自然帶出
